# DIV Tool Rule

## Tool 구조 (5 tools)

### 오케스트레이터
- `div(ticker)` — 배당 종합 (최신 상세 + 3년 추이)

### 개별 Tool
- `div_search(ticker)` — 배당 관련 공시 검색 (현금배당 결정, 중간배당)
- `div_detail(ticker, bsns_year, reprt_code)` — 배당 상세 (DPS, 총액, 성향, 수익률)
- `div_history(ticker, years)` — 배당 이력 (연도별 추이)
- `div_manual()` — 이 문서 + DIV_CASE_RULE.md 반환

## 핵심 연산 규칙

### 배당성향 (Payout Ratio)
```
배당성향(%) = 배당금 총액 / 지배주주 귀속 당기순이익 × 100
```
- **반드시 '지배주주 귀속 당기순이익' 사용** (연결재무제표 기준)
- 비지배지분 순이익을 포함하면 배당성향이 과소 계산됨
- 별도 재무제표의 당기순이익 사용 금지 (연결 vs 별도 괴리 큼)
- DART `alotMatter`에 배당성향이 있으면 그 값을 우선 사용

### 배당수익률 (Dividend Yield)
```
배당수익률(%) = 주당배당금(DPS) / 배당기준일 종가 × 100
```
- 종가: KRX Open API 일별 시세 (`TDD_CLSPRC`)
- 배당기준일: 보통 결산일 (12월 결산 기업 = 12/31)
- DART `alotMatter`에 시가배당률이 있으면 그 값을 우선 사용
- 기준일이 비거래일이면 직전 거래일 종가 사용

### 분기/반기 배당
```
연간 DPS = 1Q DPS + 반기 DPS + 3Q DPS + 기말 DPS
```
- 각 분기 배당은 해당 분기분만 (누적이 아님)
- flow(개별 분기 DPS) + stock(연간 합산 DPS) 모두 표시
- 분기배당 여부: reprt_code 11013(1Q), 11012(반기), 11014(3Q)에 데이터 있으면 해당

### 특별배당
- 정기배당과 별도. "특별" 키워드로 감지
- 연간 DPS 합산에 포함하되, 별도 행으로 표시
- 배당성향 계산 시 포함 (총배당금에 특별배당 포함)

### 주식배당
- 데이터 관리하되 현금배당 대비 중요도 낮음
- 별도 행으로 표시, DPS 합산에는 미포함

## 데이터 소스

| 데이터 | 소스 | API/필드 |
|--------|------|----------|
| 주당배당금(DPS) | DART | `alotMatter` → 주당 현금배당금 |
| 배당금 총액 | DART | `alotMatter` → 배당금총액 |
| 배당성향 | DART (우선) | `alotMatter` → 배당성향(%) |
| 배당성향 (계산) | DART | 배당금총액 / 지배주주귀속 당기순이익 |
| 시가배당률 | DART (우선) | `alotMatter` → 시가배당률(%) |
| 배당수익률 (계산) | KRX | DPS / KRX 종가 × 100 |
| 종가 | KRX | `stk_bydd_trd` → `TDD_CLSPRC` |
| 지배주주 당기순이익 | DART | 사업보고서 연결 IS 또는 `agm_financials_xml` |

## API 호출

| Tool | DART API | KRX API |
|------|----------|---------|
| div_search | search_filings × 1 | - |
| div_detail | alotMatter × 1 | - |
| div_history | alotMatter × (years × 4) | stk_bydd_trd × years |
| div (종합) | div_detail + div_history | div_history 포함 |

주의: div_history는 연도당 최대 4회 DART API 호출 (기말+3분기). 3년 = 최대 12회.
