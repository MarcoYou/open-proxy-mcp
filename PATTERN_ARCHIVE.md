## LLM fallback 설계

LLM fallback은 소프트/하드 패턴으로 해결 못 한 케이스를 처리한다.
**이 아카이브의 하드패턴을 참고하여** LLM 프롬프트에 예시로 포함하면 정확도가 높아진다.

### LLM fallback 프롬프트 설계
```
당신은 DART 공시 문서의 경력 데이터를 정리하는 전문가입니다.
아래 텍스트에서 각 경력 항목을 분리해주세요.

규칙:
- 회사명 + 직책이 하나의 항목입니다
- (주), ㈜ 는 회사명의 일부입니다
- 영문 괄호(CEO, CFO, Officer 등) 뒤에 한글이 오면 다음 항목입니다
- 대기업 그룹명(한화, 삼성, LG, SK 등) 앞에서 분리합니다

예시 (하드패턴 아카이브 참고):
입력: "한화솔라원 CSO(Chief Strategic Officer)한화큐셀 CSO..."
출력:
1. 한화솔라원 CSO(Chief Strategic Officer)
2. 한화큐셀 CSO(Chief Strategic Officer)
...
```

### 적용 시점
- `_clean_career_details` 후 content가 여전히 80자 초과
- 소프트+하드 패턴 적용 후에도 분리 안 된 케이스
- LLM API 호출 → 구조화된 결과로 대체

### 공통 원인
- 파서가 최소 필드만 반환 (`name`, `roleType`)하면 FE에서 나머지 필드(`careerDetails`, `eligibility` 등)가 undefined
- FE 컴포넌트에서 optional chaining(`?.`) 없이 접근
