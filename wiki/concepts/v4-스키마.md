# v4 스키마

## 개념

[[OpenProxy-AI]] [[파이프라인-아키텍처|파이프라인]]이 생성하는 통합 JSON의 현재 버전. KOSPI 200 (199개) 기업의 주총 소집공고 + 투표결과를 하나의 파일로 통합.

## 최상위 구조

```json
{
  "schemaVersion": "v4",
  "meetingInfo": { ... },
  "agendas": [ ... ],
  "voteResults": { ... } | null
}
```

## meetingInfo

| 필드 | 설명 |
|------|------|
| companyName | 기업명 |
| meetingDate | 주주총회 개최일 (v4 추가) |
| noticeDate | 소집공고 공시일 (v4 추가) |
| rceptNo | DART 접수번호 |

## agendas[]

각 안건 객체는 `keyData`를 포함하며, 안건 유형에 따라 다음 하위 구조를 가짐:

- **candidates** - 이사/감사위원 후보 정보 (경력, 추천인 등)
- **compensation** - [[보수한도]] (v4 추가: 전기 실지급액, [[소진율]])
- **charterChanges** - [[정관변경]] 하위 안건
- **financialStatements** - 재무제표 승인 데이터
- **treasuryStock** - [[자사주]] 처분/소각 (v4 추가)

## voteResults

```json
{
  "items": [ ... ],
  "attendance": { ... }
}
```

- `items`: 안건별 찬성/반대/기권 비율 ([[KRX-KIND]] 크롤링 기반)
- `attendance`: [[참석률]] 역산 데이터
- 주총 미개최 또는 결과 미공시 시 `null`

## v3 -> v4 차이

| 항목 | v3 | v4 |
|------|----|----|
| compensation | 한도만 | 한도 + 전기실지급 + 소진율 |
| treasuryStock | 없음 | 자사주 처분/소각 안건 구조화 |
| voteResults | 없음 | KIND 투표결과 + 참석률 통합 |
| noticeDate | 없음 | meetingInfo에 추가 |
| meetingDate | 없음 | meetingInfo에 추가 |

## 파일 네이밍

```
A{code}_v4_parsed_{name}.json
```

예: `A005930_v4_parsed_삼성전자.json`
