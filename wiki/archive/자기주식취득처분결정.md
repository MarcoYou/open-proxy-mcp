## 개요

기업이 자기주식(자사주)의 취득, 처분, 소각 등을 결정한 후 공시하는 주요사항보고. [[자사주]] 이벤트 추적의 핵심.

## 소스

- **제출처**: [[DART-OpenAPI]] (DS005 주요사항보고)
- **의무 여부**: 의무 (자사주 취득/처분 결정 시)
- **공시 주기**: 수시 (이벤트 발생 시)

## 주요 내용

- 취득/처분 결정 내용
- 주식 수량 및 금액
- 취득/처분 방법 (장내매매, 신탁 등)
- 취득/처분 기간
- 목적 (주주환원, 경영권 방어, 임직원 보상 등)

## 전체 문서 구조 (트리)

```
자기주식 취득/처분 결정
├── 결정 개요
│   ├── 취득/처분 구분
│   ├── 취득 예정 주식수 및 금액
│   └── 취득 방법 (장내매수/신탁/공개매수 등)
├── 취득 목적
│   ├── 주가 안정
│   ├── 임직원 상여 (스톡옵션)
│   └── 소각 예정 여부
└── 일정
    ├── 취득 기간
    └── 이사회 결의일
```

API: DART DS004 자기주식 취득/처분/신탁체결/해지 → own_treasury_tx 소스

## API 필드 vs 원문 대응

| API 엔드포인트 | 공시 유형 | 주요 필드 | 주의사항 |
|----------------|-----------|-----------|----------|
| `tsstkAqDecsn` | 취득 결정 | 취득 주식수, 취득 금액, 취득 방법, 취득 기간 | |
| `tsstkDpDecsn` | 처분 결정 | 처분 주식수, 처분 금액, 처분 방법 | |
| `tsstkAqTrctrCnsDecsn` | 신탁계약 체결 | 신탁 금액, 신탁 기간 | 실제 취득은 별도 공시로 나옴 |
| `tsstkAqTrctrCcDecsn` | 신탁계약 해지 | 해지 사유, 미취득 잔액 | |

**결정 공시 ≠ 실제 취득 완료** — 취득 기간 중 실제 매수 내역은 별도 체결 보고에서 확인.

## 파싱 주의사항

- 신탁 방식의 경우 체결/해지 각각 공시 — 실제 취득 수량은 해지 공시에서 확인
- 소각 예정 여부는 유통주식 수 변동과 직결 — [[지분구조]] 분석 시 반드시 반영
- 취득 결정 후 미취득(취소) 사례 있음 — 결정 공시만으로 실제 취득 수량 단정 금지

## OPM에서의 활용

own_treasury_tx tool이 [[DART-OpenAPI]] DS005의 4개 API를 사용:
- tsstkAqDecsn: 취득 결정
- tsstkDpDecsn: 처분 결정
- tsstkAqTrctrCnsDecsn: 신탁계약 체결 결정
- tsstkAqTrctrCcDecsn: 신탁계약 해지 결정

[[own-tool-rule]]에서 수시 공시 소스(우선순위 2위)로 활용.

## 관련

[[자사주]] 보유 현황은 [[사업보고서]]의 own_treasury에서 확인. 이벤트성 변동은 이 공시에서 추적. [[지분구조]]의 유통주식 변동에 직접 영향.
