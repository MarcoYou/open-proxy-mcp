# 단일판매·공급계약 체결 / 해지

## 개요

회사가 **최근매출액 5% 이상** 규모의 단일 판매·공급 계약을 체결하거나 해지할 때 제출하는 의무 공시. 대형 수주 뉴스이자 **거래 상대방이 특수관계인이면 일감몰아주기 핵심 신호**.

## 공시 제목 변형

| 제목 | pblntf_ty | 구분 |
|------|-----------|------|
| 단일판매ㆍ공급계약체결 | I | 거래소공시 (의무) |
| 단일판매ㆍ공급계약체결(자율공시) | I | 자율 (기준 미달이지만 자발) |
| 단일판매ㆍ공급계약체결(자회사의주요경영사항) | I | 모회사 명의 자회사 거래 |
| 단일판매ㆍ공급계약해지 | I | 의무 |
| 단일판매ㆍ공급계약해지(자회사의주요경영사항) | I | 자회사 해지 |
| [기재정정]... | I | 정정공시 |

## 소스

- **제출처**: [[DART-OpenAPI]] (+ [[KRX-KIND]] 병행 제출)
- **공시 유형**: pblntf_ty=I (거래소공시)
- **DART API**: 전용 구조화 API 없음. `list.json` + 키워드
- **의무 여부**: 의무 (최근매출 5% 이상, 회사 규모별 기준 다름)
- **공시 주기**: 수시 (계약 체결/해지 발생 즉시)

## API 필드 vs 원문 대응

list.json 기본 필드만 제공. 거래 상세는 원문 파싱 필요.

| API 필드 | 의미 | 비고 |
|----------|------|------|
| `rcept_no`, `rcept_dt` | 접수번호/일 | |
| `corp_code`, `corp_name` | 공시 회사 | |
| `report_nm` | 제목 | 체결/해지 + 자율/자회사 구분 |
| `flr_nm` | 제출인 | 자회사 공시 시 모회사명 |

**원문 파싱 필수 항목**:
- 계약 상대방명 및 특수관계 여부
- 계약금액
- 최근매출 대비 비율

## OPM에서의 활용

- **`related_party_transaction(scope="supply_contract")`**: list.json 메타만 수집. 방향(체결/해지), 자회사/자율 플래그

## 거버넌스 분석 포인트

### 일감몰아주기 시그널 (원문 파싱 필요)

1. **계약 상대방이 특수관계인**:
   - 동일 그룹 계열사 (대기업집단 내부거래)
   - 대주주 지배회사
   - 임원·주요주주 소유 회사
2. **반복 체결 패턴**:
   - 매 분기·반기 동일 거래처로 계약 체결
   - 단일 거래처 매출 의존도 > 50%
3. **자회사 주요경영사항 공시**:
   - 모회사가 비상장 자회사의 거래를 대신 공시
   - 거래처가 그룹 내 다른 계열사면 그룹 내부거래

### 업종별 해석

- **건설업**: 빈번한 계약 당연 (현대건설 2년 72건 사례). 그룹 계열사 비중 관찰.
- **반도체**: 대형 고객사 수주 → 매출 가시성
- **섬유·화학**: 대형 제조사 거래 — 특수관계 여부 판별 필요

### 계약 해지

- 정상적 만기 종료: 중립
- 조기 해지: 부정 신호 (상대방 재무 악화, 분쟁)
- 해지 공시 빈도 높으면 계약 품질 의심

### 자율공시 (`(자율공시)`)

- 의무 기준(매출 5%) 미달이지만 공시 → 투자자 정보 제공 의지 (긍정)
- 계열사 계약을 자율 공시로 올려 투명성 확보 사례
