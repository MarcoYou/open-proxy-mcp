# Case Definitions — AGM 파서 성공 기준

## 목적

이 문서는 두 가지 역할을 합니다:

1. **LLM fallback 가이드** — 각 안건이 뭔지, 성공적으로 파싱하면 어떻게 생겨야 하는지의 도메인 설명 + 실제 예시. LLM이 이 문서를 참고하여 예시에 가깝게 파싱하도록 유도.
2. **기계적 품질 판정** — SUCCESS / SOFT_FAIL / HARD_FAIL 자동 분류 기준. PDF fallback과 LLM fallback 트리거 조건.

## Fallback Decision Matrix

```
파서 결과 판정
  ├─ SUCCESS → 그대로 사용
  ├─ SOFT_FAIL → PDF fallback으로 보강 시도
  │   └─ 여전히 SOFT_FAIL → LLM fallback (이 문서의 예시를 few-shot으로 전달)
  └─ HARD_FAIL → PDF fallback 시도
      └─ 여전히 HARD_FAIL → LLM fallback (이 문서의 예시를 few-shot으로 전달)
```

## 1. agm_agenda — 안건 목록

### 이 안건은 무엇인가

주주총회 소집공고의 **회의목적사항** 섹션에서 추출하는 안건 트리입니다. 모든 소집공고에 반드시 존재합니다.

- 제1호, 제2호, ... 형식의 안건 번호
- 세부의안이 있으면 제2-1호, 제2-2호, 제3-1-1호 형식으로 하위 트리 구성
- 각 안건에는 "재무제표 승인", "이사 선임", "정관변경" 등의 제목이 붙음
- 일반적으로 정기주총은 3-8개 안건, 대기업은 5-12개 안건

### 성공 파싱 예시

삼성전자 2026년 정기주총:

```json
[
  {
    "number": "제1호",
    "title": "정관 일부 변경의 건",
    "children": [
      {"number": "제1-1호", "title": "집중투표제 배제 조항 삭제", "children": []},
      {"number": "제1-2호", "title": "개정 상법 반영", "children": []},
      {"number": "제1-3호", "title": "이사의 임기 조문 정비", "children": []},
      {"number": "제1-4호", "title": "주식 소각 조문 정비", "children": []}
    ]
  },
  {
    "number": "제2호",
    "title": "제57기(2025.1.1. ~ 2025.12.31.) 재무상태표, 손익계산서 및 이익잉여금처분계산서 등 재무제표 승인의 건",
    "children": []
  },
  {
    "number": "제3호",
    "title": "사내이사 김용관 선임의 건",
    "children": []
  }
]
```

**핵심 포인트:**
- `number`는 항상 "제N호" 또는 "제N-M호" 형식
- 드물게 "제N-M-J호" 형식의 `number`도 존재함
- `title`은 안건의 핵심 내용만 포함 (테이블, 조문 텍스트 등이 섞이면 안 됨)
- 세부의안은 `children` 배열에 재귀적으로 포함

### 판정 기준

| 등급 | 기준 |
|------|------|
| **SUCCESS** | items >= 1, 모든 title 2-150자, number가 `제N호`/`제N-M호` 패턴, 중복 number 없음 |
| **SOFT_FAIL** | title 150-200자 (zone 텍스트 혼입 의심), title에 `\|` 포함 (테이블 내용 혼입), 안건 1개만 (정기주총인데 비정상) |
| **HARD_FAIL** | items 0개, title > 200자, 중복 number, 빈 title |

## 2. agm_financials — 재무제표

### 이 안건은 무엇인가

재무제표 승인 안건의 상세 내용입니다. 정기주총의 제1호 안건인 경우가 많습니다. 주주총회 결의 사항이 아닌 경우 단순 보고사항인 경우도 있습니다.

- **재무상태표(Balance Sheet)**: 자산/부채/자본의 기말 잔액. 당기/전기 비교.
- **손익계산서(Income Statement)**: 매출/비용/이익 흐름. 당기/전기 비교.
- 연결재무제표(consolidated)와 별도재무제표(separate)가 각각 있을 수 있음
- 단위: 천원, 백만원, 억원 중 하나
- 핵심 계정: 자산총계, 부채총계, 자본총계 (BS) / 매출액(또는 영업수익), 영업이익, 당기순이익 (IS)

### 성공 파싱 예시

삼성전자 2026년 정기주총 (연결, 재무상태표 앞부분):

```json
{
  "consolidated": {
    "balance_sheet": {
      "unit": "백만원",
      "columns": ["account", "current", "prior"],
      "rows": [
        ["Ⅰ. 유동자산", "247,684,612", "227,062,266"],
        ["1. 현금및현금성자산", "57,856,378", "53,705,579"],
        ["2. 단기금융상품", "67,965,021", "58,909,334"],
        ["...(중략)..."],
        ["자산총계", "515,123,456", "478,901,234"]
      ]
    },
    "income_statement": {
      "unit": "백만원",
      "rows": [
        ["Ⅰ. 매출액", "333,605,938", "300,870,903"],
        ["Ⅱ. 매출원가", "202,235,513", "186,562,268"],
        ["Ⅲ. 매출총이익", "131,370,425", "114,308,635"],
        ["판매비와관리비", "87,769,374", "81,582,674"],
        ["Ⅳ. 영업이익", "43,601,051", "32,725,961"]
      ]
    }
  }
}
```

**핵심 포인트:**
- `unit`이 반드시 있어야 함 (숫자 해석 불가능)
- `current`/`prior` 컬럼으로 당기/전기 비교
- 숫자는 쉼표 포함 문자열 (예: "247,684,612")
- BS에 자산총계/부채총계/자본총계, IS에 매출액/영업이익이 반드시 포함

### 판정 기준

| 등급 | 기준 |
|------|------|
| **SUCCESS** | BS rows >= 5 + IS rows >= 3, unit 존재, 핵심 계정 포함 (자산총계/자산합계, 매출액/영업수익/영업이익), 50%+ 셀이 숫자 |
| **SOFT_FAIL** | BS만 있고 IS 없음 (또는 반대), rows < 기준, 핵심 계정 누락, 빈 셀 30%+ |
| **HARD_FAIL** | BS/IS 모두 None, rows 0개, unit 없음, 숫자 비율 20% 미만 (파싱 쓰레기) |

## 3. agm_personnel — 이사/감사 선임

### 이 안건은 무엇인가

이사/감사 선임·해임 안건입니다. 후보자별로 다음 정보를 DART 표준 서식의 테이블에서 추출합니다:

- **성명 + 생년월일**: 후보자 식별
- **직위/카테고리**: 사내이사, 사외이사, 감사위원, 감사 등
- **추천인**: 이사회, 사외이사후보추천위원회 등
- **세부경력(careerDetails)**: 가장 중요한 필드. 각 경력은 **기간 + 내용(회사명+직책)** 으로 분리되어야 함
- **결격사유**: 체납/부실기업/법령상 결격 여부
- **직무수행계획**: 선임 후 어떻게 기여할지
- **추천사유**: 왜 이 후보자를 추천하는지

### 성공 파싱 예시

KT&G 2026년 정기주총, 사외이사 노환용:

```json
{
  "number": "제4호",
  "title": "사외이사 노환용 선임의 건",
  "action": "선임",
  "category": "사외이사",
  "candidates": [{
    "name": "노환용",
    "birthDate": "1956.4.25",
    "roleType": "사외이사",
    "recommender": "KT&G 사외이사후보추천위원회",
    "mainJob": "LG상록재단 비상임이사",
    "careerDetails": [
      {"period": "2021~현재", "content": "現) LG상록재단 비상임이사"},
      {"period": "2022~2024", "content": "前) 경상남도 투자유치자문위원"},
      {"period": "2016~2021", "content": "前) LG연암학원 비상임이사"},
      {"period": "2015~2020", "content": "前) LG전자 상근고문"},
      {"period": "2014~2015", "content": "前) LG전자 B2B부문장, 사장"},
      {"period": "2010~2014", "content": "前) LG전자 AE사업본부장, 사장"}
    ],
    "eligibility": {
      "taxDelinquency": "해당사항 없음",
      "insolventMgmt": null,
      "legalDisqualification": null
    }
  }]
}
```

**핵심 포인트:**
- `name`은 한글 2-5자 (영문 병기 시 10자 이내). 절대 조문번호("제31조...")나 안건번호("제2-1호")가 아님
- `careerDetails` 각 항목의 `content`는 **한 줄에 하나의 직책**. 50자 이내가 정상. 여러 경력이 줄바꿈 없이 합쳐져 100자+ 되면 SOFT_FAIL
- `period`는 "YYYY-YYYY" 또는 "YYYY.MM-YYYY.MM" 또는 "YYYY-현재" 형식
- 경력이 0건이면 파싱 실패 (DART 원본에는 반드시 경력이 있음)

### 실패 예시 (이렇게 되면 안 됨)

**career 병합 (SOFT_FAIL):**
```json
{
  "content": "한화솔라원 CSO한화큐셀 CSO한화솔라원 CCO한화큐셀 CCO㈜한화 전략부문장..."
}
```
→ 516자짜리 한 줄. 원래 13개 경력이 분리되어야 함.

**이름이 조문번호 (HARD_FAIL):**
```json
{
  "name": "제 31 조 (이사의 선임) 이사는 주주총회에서 선임한다..."
}
```
→ 정관 텍스트가 이름으로 잡힌 케이스.

### 판정 기준

| 등급 | 기준 |
|------|------|
| **SUCCESS** | appointments >= 1, 모든 name 2-10자이고 한글/영문 이름 패턴, 모든 candidate에 careerDetails >= 1건, 각 content <= 100자 |
| **SOFT_FAIL** | careerDetails 빈 배열, content > 100자 (경력 병합), birthDate 누락, 경력 1건만 (의심) |
| **HARD_FAIL** | appointments 0개, name이 `제N조` 또는 `제N호` 패턴 (조문/안건번호 혼입), name > 20자, 모든 후보자 careerDetails 없음 |

## 4. agm_aoi_change — 정관변경

### 이 안건은 무엇인가

정관변경 안건입니다. 변경하려는 조항별로 **현행(변경전)**, **변경후**, **변경 사유**를 비교 테이블로 표시합니다.

- 세부의안(제3-1호, 제3-2호 등)으로 나뉘기도 함
- 2025-2026년에는 상법 개정(집중투표제, 이사 충실의무 등)으로 거의 모든 기업이 정관변경 안건 포함
- 변경전 텍스트에는 기존 조문 전문, 변경후에는 수정된 조문 전문 또는 "<삭제>" 표시

### 성공 파싱 예시

KT&G 2026년 정기주총, 집중투표 규정 정비:

```json
{
  "amendments": [{
    "subAgendaId": "2-4",
    "label": "집중투표 규정 정비",
    "clause": "제26조(사장 및 이사의 선임 등)",
    "before": "제26조(사장 및 이사의 선임 등) ⑥ 집중투표의 방법에 의하여 이사를 선임하는 경우 대표이사 사장과 그 외의 이사를 별개의 조로 구분한다.",
    "after": "제26조(사장 및 이사의 선임 등) ⑥ <삭 제>",
    "reason": "집중투표규정정비"
  }],
  "summary": {
    "totalAmendments": 1
  }
}
```

**핵심 포인트:**
- `before`와 `after` 모두 실제 조문 텍스트여야 함 (빈 문자열이나 `"---"` placeholder가 아님)
- `clause`에 `제N조` 형식의 조항 참조가 있어야 함
- `reason`에 변경 사유가 있어야 함 (없을 수도 있지만 대부분 있음)
- `<삭제>`는 정상 (조항 삭제 의미)

### 참고: `---` 생략 표기는 정상

DART 정관변경 표준 서식에서 변경 없는 부분은 `------`으로 생략 표기하고, 달라진 부분만 실제 텍스트로 표시합니다. 예:

```json
{
  "before": "④ 감사위원회 위원은 주주총회에서 이사를 선임한 후 선임된 이사 중에서 감사위원을 선임하여야 한다. 이 경우 감사위원회 위원 중 1명은 주주총회 결의로 다른 이사들과 분리하여 감사위원회 위원이 되는 이사로 선임하여야 한다.",
  "after": "④ -------------------------------------------------- 2명 ------------------------------------------------."
}
```

이것은 **정상**입니다. "1명 → 2명"만 변경된 것이고, 나머지는 동일하므로 `---`로 생략한 것.

**진짜 실패**는 before/after가 완전히 빈 문자열이거나, 조문이 아닌 쓰레기 텍스트(테이블 헤더 등)가 들어간 경우입니다.

### 판정 기준

| 등급 | 기준 |
|------|------|
| **SUCCESS** | amendments >= 1, 모든 before/after가 비어있지 않고 5자+, clause에 `제N조` 포함 |
| **SOFT_FAIL** | clause 누락, reason 누락, before == after (변경 없음?), subAgendaId 누락, before/after > 2000자 (zone leak 의심) |
| **HARD_FAIL** | amendments 0개 (정관변경 안건이 있는데), before/after 모두 빈 문자열 |

## 5. agm_compensation — 보수한도

### 이 안건은 무엇인가

이사/감사 보수한도 승인 안건입니다. 거의 모든 정기주총에 존재합니다 (KOSPI 200 중 96%).

DART 표준 서식으로:
- **당기**: 이사의 수(사외이사수) + 보수 최고한도액
- **전기**: 이사의 수(사외이사수) + 실제 지급된 보수총액 + 최고한도액

이사와 감사가 별도 안건인 기업도 있고 (GKL 등), 합산인 기업도 있습니다.

### 성공 파싱 예시

삼성전자 2026년 정기주총:

```json
{
  "items": [{
    "number": "제5호",
    "title": "이사 보수한도 승인의 건",
    "target": "이사",
    "current": {
      "headcount": "8(5)",
      "totalDirectors": 8,
      "outsideDirectors": 5,
      "limit": "450억원",
      "limitAmount": 45000000000
    },
    "prior": {
      "headcount": "10(6)",
      "totalDirectors": 10,
      "outsideDirectors": 6,
      "actualPaid": "287억원",
      "actualPaidAmount": 28700000000,
      "limit": "360억원",
      "limitAmount": 36000000000
    },
    "notes": ["※ 당기(제58기) 보수 한도 총액 450억 : 일반보수 260억, 장기성과보수 190억"]
  }],
  "summary": {
    "totalItems": 1,
    "currentTotalLimit": 45000000000,
    "priorTotalPaid": 28700000000,
    "priorTotalLimit": 36000000000,
    "priorUtilization": 79.7
  }
}
```

**핵심 포인트:**
- `current.limitAmount`가 양수 정수 (원 단위)여야 함 — 이 숫자가 핵심
- `headcount`에 이사의 수와 사외이사수가 "N(M)" 형식으로 포함
- `prior`에 실제 지급액과 한도가 있어야 전기 대비 소진율 계산 가능
- `target`은 "이사" 또는 "감사"

### 판정 기준

| 등급 | 기준 |
|------|------|
| **SUCCESS** | items >= 1, 모든 item에 current.limitAmount > 0, current.headcount 존재, prior에 limitAmount 또는 actualPaidAmount 중 하나 이상 |
| **SOFT_FAIL** | current 있지만 prior가 빈 dict (데이터 부분 추출), headcount 있지만 totalDirectors 미파싱, summary.priorUtilization이 None |
| **HARD_FAIL** | items 0개 (보수한도 안건이 있는데), 모든 item의 current가 빈 dict, limitAmount가 전부 None |
